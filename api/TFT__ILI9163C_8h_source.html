00001 /*
00002     ILI9163C - A fast SPI driver for TFT that use Ilitek ILI9163C.
00003     
00004     Features:
00005     - Very FAST!, expecially with Teensy 3.x where uses DMA SPI.
00006     - It uses just 4 or 5 wires.
00007     - Compatible at command level with Adafruit display series so it's easy to adapt existing code.
00008     - It uses the standard Adafruit_GFX Library (you need to install). 
00009     
00010     Background:
00011     I got one of those displays from a chinese ebay seller but unfortunatly I cannot get
00012     any working library so I decided to hack it. ILI9163C looks pretty similar to other 
00013     display driver but it uses it's own commands so it's tricky to work with it unlsess you
00014     carefully fight with his gigantic and not so clever datasheet.
00015     My display it's a 1.44"", 128x128 that suppose to substitute Nokia 5110 LCD and here's the 
00016     first confusion! Many sellers claim that it's compatible with Nokia 5110 (that use a philips
00017     controller) but the only similarity it's the pin names since that this one it's color and
00018     have totally different controller that's not compatible.
00019     http://www.ebay.com/itm/Replace-Nokia-5110-LCD-1-44-Red-Serial-128X128-SPI-Color-TFT-LCD-Display-Module-/141196897388
00020     http://www.elecrow.com/144-128x-128-tft-lcd-with-spi-interface-p-855.html
00021     Pay attention that   can drive different resolutions and your display can be
00022     160*128 or whatever, also there's a strain of this display with a black PCB that a friend of mine
00023     got some weeks ago and need some small changes in library to get working.
00024     If you look at TFT_ILI9163C.h file you can add your modifications and let me know so I
00025     can include for future versions.
00026     
00027     Code Optimizations:
00028     The purpose of this library it's SPEED. I have tried to use hardware optimized calls
00029     where was possible and results are quite good for most applications, actually nly filled circles
00030     are still a bit slow. Many SPI call has been optimized by reduce un-needed triggers to RS and CS
00031     lines. Of course it can be improved so feel free to add suggestions.
