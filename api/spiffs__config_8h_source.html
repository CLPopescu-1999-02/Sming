00195 // Enable this to add a temporal file cache using the fd buffer.
00196 // The effects of the cache is that SPIFFS_open will find the file faster in
00197 // certain cases. It will make it a lot easier for spiffs to find files
00198 // opened frequently, reducing number of readings from the spi flash for
00199 // finding those files.
00200 // This will grow each fd by 6 bytes. If your files are opened in patterns
00201 // with a degree of temporal locality, the system is optimized.
00202 // Examples can be letting spiffs serve web content, where one file is the css.
00203 // The css is accessed for each html file that is opened, meaning it is
00204 // accessed almost every second time a file is opened. Another example could be
00205 // a log file that is often opened, written, and closed.
00206 // The size of the cache is number of given file descriptors, as it piggybacks
00207 // on the fd update mechanism. The cache lives in the closed file descriptors.
00208 // When closed, the fd know the whereabouts of the file. Instead of forgetting
00209 // this, the temporal cache will keep handling updates to that file even if the
00210 // fd is closed. If the file is opened again, the location of the file is found
00211 // directly. If all available descriptors become opened, all cache memory is
00212 // lost.
00213 #ifndef SPIFFS_TEMPORAL_FD_CACHE
00214 #define SPIFFS_TEMPORAL_FD_CACHE              1
00215 #endif
00216 
00217 // Temporal file cache hit score. Each time a file is opened, all cached files
00218 // will lose one point. If the opened file is found in cache, that entry will
00219 // gain SPIFFS_TEMPORAL_CACHE_HIT_SCORE points. One can experiment with this
00220 // value for the specific access patterns of the application. However, it must
00221 // be between 1 (no gain for hitting a cached entry often) and 255.
00222 #ifndef SPIFFS_TEMPORAL_CACHE_HIT_SCORE
00223 #define SPIFFS_TEMPORAL_CACHE_HIT_SCORE       4
00224 #endif
